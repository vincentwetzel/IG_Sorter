# Project Architecture

The **IG_Sorter** project is designed as a collection of procedural Python scripts that operate on a local file system. The architecture is primarily focused on file I/O operations, text processing (Regex), and basic database interactions (Excel/Text files).

## High-Level Workflow

1. **Input**: Image files are manually placed into a staging area (`NEED TO SORT` directories).
2. **Processing**: `Instagram_Sorter.py` scans the staging area.
3. **Identification**:
   - The script uses Regular Expressions (Regex) to extract usernames or source types from the downloaded file names.
   - It cross-references these extracted names against a local database (`ig_people.xlsx`) using the `pandas` library.
4. **Action**:
   - Known files are renamed sequentially and moved to their final destination directories (`SFW`, `MSFW`, `NSFW`).
   - Unknown files trigger an interactive fallback mechanism.
5. **Maintenance**: Periodically, `Find_Duplicate_Files.py` is run to identify redundancies in the destination directories.

## Core Components

### 1. Data Storage
- **`ig_people.xlsx`**: A local Excel spreadsheet used as a simple relational database. It maps `Account` (Instagram handle) to `Name` (Real-life name). Managed via the `pandas` and `openpyxl` libraries.
- **`photographers.txt`**: A flat text file containing a simple list of strings representing Instagram handles of photographers. Used to separate artist works from personal photos.

### 2. Processing Logic
- **Regex Engines**: Extensively used in `Instagram_Sorter.py` (`get_ig_name_from_filename`) to parse complex filenames generated by various third-party downloaders (Chrome extensions, Android apps, etc.).
- **File System Operations**: Handled via the built-in `os` and `subprocess` modules to move files, rename files, and open Windows Explorer windows.

### 3. External Integrations
- **Web Browser Integration**: The `webbrowser` module is used to automatically open Instagram profile pages for unknown accounts, aiding the user in manual identification.
- **Instaloader Integration**: `Instaloader_Login_Error_Fixer.py` directly interacts with local Firefox SQLite databases (`sqlite3`) to extract session cookies and injects them into the `instaloader` Python library's session object.

## Directory Structure Considerations
The scripts currently rely on hardcoded absolute paths pointing to a specific Google Drive directory structure. Users will need to update the `ROOT_PICTURE_DIR` variable to match their local environment.
